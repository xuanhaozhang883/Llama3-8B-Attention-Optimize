# Reproducibility Guide

This is the lookup document for running the current repository on a new machine. The rule is simple: clone the entire repository, preserve the directory structure, and start Python or simulator commands from the repository root unless a module README explicitly says otherwise.

## Standard Data Sources

| Consumer | Default source | Generated files | PASS evidence |
| --- | --- | --- | --- |
| RoPE RTL | `golden_model_outputs/fpga_slice/q_before_rope.npy`, `k_before_rope.npy`, `q_after_rope.npy`, `k_after_rope.npy` | `RoPE/data/*.hex` | portable Golden TB; current RTL correctly reports FAIL |
| Mask HLS | `golden_model_outputs/fpga_slice/scores_before_mask.npy`, `scores_after_mask.npy` | `attention_mask_module/mask_test_vectors/*.hex` | bit-exact file TB PASS |
| Softmax RTL | `golden_model_outputs/fpga_slice/scores_after_mask.npy`, `softmax_weights.npy` | `softmax_module/data/*.mem` | `TEST_RESULT: PASS` |
| CPU baseline | deterministic synthetic Q/K/V | `CPU_Baseline/results/<run>/` | `PASS: CPU performance benchmark finished` |

The `.npy` containers are float32, but Golden Model values have already been rounded to BF16 precision at hardware module boundaries. RTL/HLS vectors carry BF16 raw 16-bit words.

> RoPE status: the testbench is portable and self-checking, but the current BF16 arithmetic RTL does not yet meet the shared Golden tolerance. Its expected present result is `TEST_RESULT: FAIL`, with the documented mismatch counts in `RoPE/README.md`. This is a design issue, not a path issue or a testbench issue.

## Reproduction Checklist

1. Confirm expected files exist:

   ```text
   golden_model_outputs/fpga_slice/
   RoPE/sin_bf16_all.hex
   RoPE/cos_bf16_all.hex
   ```

2. Prepare module vectors. These commands use portable defaults and need no personal absolute paths:

   ```bash
   python3 RoPE/tools/prepare_rope_vectors.py
   python3 attention_mask_module/tools/convert_mask_pair_to_hex.py
   python3 softmax_module/scripts/prepare_softmax_golden.py
   ```

3. Run the desired module testbench.

   ```text
   RoPE: source RoPE/run_rope_sim.tcl in the Vivado Tcl Console.
   Mask: create/open the Vitis HLS component and use the C Simulation flow.
   Softmax: start xsim at repo root; tb_file_paths.svh uses relative paths.
   CPU: cd CPU_Baseline, then python -m cpu_baseline.self_test.
   ```

## Path Contract

| Location | Why it must stay here |
| --- | --- |
| `golden_model_outputs/fpga_slice/` | Shared module Golden inputs and expected results |
| `RoPE/data/` | Reproducible BF16 input/golden vectors for the RoPE file testbench |
| `RoPE/results/` | RoPE simulation outputs; not versioned because they can be regenerated |
| `attention_mask_module/mask_test_vectors/` | BF16 raw score input and expected masked score vectors |
| `softmax_module/data/` | `.mem` input, mask, and expected-probability vectors |
| `softmax_module/results/` | CSV comparison reports regenerated by simulation |

No source or testbench should refer to an individual user's `C:/Users/...`, `/Users/...`, or `/mnt/data/...` directory.

## If a File Cannot Be Opened

1. First run the corresponding Python preparation script. It prints the output directory on success.
2. Verify that the simulator was started through `RoPE/run_rope_sim.tcl` or from repository root.
3. For custom layouts, pass the RoPE `ROPE_*` plusargs listed in [RoPE/README.md](../RoPE/README.md). Use forward slashes on Windows.
4. Do not replace missing Golden data with random inputs when verifying correctness: that only tests connectivity, not correctness against the shared reference.
